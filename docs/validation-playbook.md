# Validation Playbook

## Purpose

这个文档定义第一阶段如何验证“正确发布 HDR”。

重点不是理论讨论，而是结果判断。

## Primary Devices

当前基线设备：

- DJI Pocket 系列
- iPhone Pro 系列
- MacBook Pro 作为主要监看设备

## Primary Questions

每次验证优先看下面几个问题：

- 上传或导出后是否发灰
- 天空是否发白
- 高光是否仍有层次
- 色彩层次是否被压塌
- MacBook 与手机观感差异是否可解释

## Minimum Validation Scenes

建议优先准备这些场景：

- Pocket 高光场景
- Pocket 天空场景
- iPhone 肤色场景
- iPhone 夜景高反差场景

## Evidence Format

优先保留小体积证据：

- 关键帧截图
- 导出参数记录
- 简短观察结论

截图命名规则：

- `<device>_<scene>_<variant>_<date>.png`

## Current Release Bias

当前阶段优先关注：

- Resolve 内监看结果
- 导出后的观感
- 抖音优先链路

## What Not To Over-Optimize Yet

暂时不要把时间花在：

- 全平台全规格覆盖
- 理论上最完整的色彩体系设计
- 非当前主线设备的广泛兼容
